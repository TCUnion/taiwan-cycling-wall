# 📋 實施計畫：集合點與路線起點不一致警告

## 任務類型
- [x] 前端
- [x] 後端（小幅工具抽離）
- [ ] 全棧

## 需求（Prompt 增強後）

於「發起活動／編輯活動」頁面，當使用者：
1. 提供集合點 Google Maps 連結（`spotUrl`），且
2. 套用了路線庫路線（`已套用路線庫座標`）或編輯既有活動已帶座標

兩端皆能解析出座標時，**即時計算兩點距離**；若距離 > 1 公里，顯示**橘色非阻擋式警告**，提醒使用者集合點與路線起點可能不一致。

## 關鍵設計決策（綜合 Codex + Gemini）

| 項目 | 決策 | 理由 |
|------|------|------|
| 警告位置 | inline 區塊，緊接於既有 `提交錯誤` 區塊下方 | 主流程內可見、不打斷送出 |
| 觸發時機 | 即時（`useMemo`），不等到送出 | 使用者改完欄位立刻看到、可即時修正 |
| 閾值 | **1 公里** | Codex 建議 3km 太鬆、Gemini 0.5km 對停車場→路線起點誤報率高；台灣都市/郊區約騎場景 1km 為合理中位數 |
| 視覺風格 | flat amber 警告塊（`bg-amber-50 border border-amber-200 text-amber-800`），與既有 `提交錯誤` 紅色塊對稱 | 維持表單區塊內部視覺一致；不採用 sticky-rotate（form 區塊本身不是便利貼） |
| Icon | Lucide `AlertTriangle`（已有 import 慣例） | 不使用 emoji（CLAUDE.md 規範） |
| 動作按鈕 | **無修正按鈕，只資訊顯示** + 「我知道了」忽略按鈕（一次性 dismiss） | 集合點與路線起點本來就可能合理不同（例如停車場停在 1km 外），自動覆蓋會破壞使用者刻意設定 |
| Phase 2 | 擴充 `/api/route-info` 回傳起點座標（Strava/RWGPS） | 第一版不做：HTML 結構不穩、誤報風險、不在驗收必要範圍 |
| 共用 utility | 將 `解析地圖座標` 從 `EventWeatherCard.tsx` / `EventWeatherInline.tsx` 抽到 `src/utils/geoUtils.ts` | 已有兩份重複，再加第三份不合理（DRY） |
| 編輯模式 | 修正 `已套用路線庫座標` 初始值為 `預填來源?.routeCoordinates ?? []` | 既有活動的 `routeCoordinates` 才能參與檢查 |
| 定期約騎 | 表單層警告即可自然適用 | 同一份 state 產生批次活動 |

**檔案**：`src/utils/geoUtils.ts`（新建）

export interface 座標 { lat: number; lon: number }

// 從 Google Maps URL 解析座標。支援 @lat,lng / !3d!4d / ?q= / ?ll= / ?q=loc:
// shortened URL（maps.app.goo.gl）回傳 null
export function 解析地圖座標(url?: string): 座標 | null

// Haversine 公式計算兩點距離（公里）
export function 計算座標距離公里(a: 座標, b: 座標): number

// 集合點 / 路線起點一致性檢查
export interface 一致性警告 {
  距離公里: number
  集合點座標: 座標
  路線起點座標: 座標
}
export function 建立集合點路線起點警告(params: {
  集合點URL?: string
  路線座標?: [number, number][]

}): 一致性警告 | null

### Step 4：單元測試

**檔案**：`src/utils/geoUtils.test.ts`（新建）

涵蓋情境：
- `解析地圖座標` 五種 URL 格式各一筆
- shortened URL（maps.app.goo.gl）回傳 null
- 緯度 > 90 / 經度 > 180 回傳 null
- Haversine 同點 = 0
- Haversine 已知兩點距離（如台北車站 25.0478,121.5170 → 板橋車站 25.0136,121.4636，約 5.4 km）
- `建立集合點路線起點警告` 無 URL 回傳 null
- 無路線座標回傳 null
- shortened URL 回傳 null
- 距離 ≤ 閾值不警告
- 距離 > 閾值回傳警告物件含距離與兩點

預期產物：`npx vitest run src/utils/geoUtils` 全部通過。

## 風險與緩解

| 風險 | 緩解 |
|------|------|
| Haversine 在小距離精度誤差 | 公里級用 Haversine 已足夠；不需要 Vincenty |
| `useMemo` 依賴陣列遺漏觸發 re-render | 依賴 `[spotUrl, 已套用路線庫座標]`，兩者都是 state，變更必觸發 |
| 使用者套用範本後 `已套用路線庫座標` 不會變 | 範本不帶路線座標（型別 `RideTemplate` 無此欄位），警告自然不觸發，預期行為 |
| 編輯 EventWeather 元件時破壞既有功能 | 只移除函式定義改 import，不動呼叫端 |
| 警告誤報（停車場停在 1km 外） | 可由使用者按「我知道了」隱藏；非阻擋；閾值常數化未來可調 |
